The two families

Encoder

Read. Label. Done.
  • BERT, RoBERTa, DeBERTa, ModernBERT
  • Reads a passage in full, returns a fixed prediction (label, score)
  • Deterministic, cheap, fast
  • Runs on a laptop or free Colab CPU
  • Good for stance, sentiment, topic, NER, any closed-label task

Generative

Continue the text.
  • Claude, GPT, Gemini, LLaMA, Qwen
  • Given a prompt, produces more text
  • Stochastic, costly per call, slower
  • Usually called via API
  • Good for extraction, summarization, free-form judgment
  • Reach for it when the answer cannot fit in a fixed label set

The NLI trick · why an encoder can do almost anything

An NLI-trained encoder learns one job. Given a premise and a hypothesis, decide entailment / neutral / contradiction. Any classification can be rewritten as a yes-or-no entailment question. zero-shot-classification does this for you with a list of candidate labels. Park (2026) aggregates 13 such hypotheses with weights into a continuous -5 to +5 stance score.

Pitfalls

  • "Looks impressive" ≠ publishable. Always validate against a hand-coded gold set.
  • Off-the-shelf trips on interpretive text. Mixed praise + criticism needs hand-labels.
  • Wording of candidate labels matters. The model reads them as hypotheses, not keywords.
  • Pin model versions. Cloud APIs retire models. HF model IDs survive.
  • Never paste API keys in code cells. Use Colab's userdata sidebar.
